CSS优先级浅析

  2011-03-03     HTML     CSS

看到很多人写过关于CSS优先级的长篇大论，也曾试图认真读完某一篇，但都没有坚持到底。可能是我的经验比较少吧，到目前为止也没碰到过那么复杂的样式应用。因为在使用层级选择器的时候我一向比较谨慎，很少使用3级以上的层级选择符，极少使用4级以上的选择符，而且样式名的命名也比较单一，不会到处都写，并且我喜欢追求风格的统一，所以没有那么多特殊的爱好给每个标签都加上一套样式。废话有点多了，这里说下本人的一点浅见。

CSS的优先级应该先看是否描述到该元素，再看最高级别的等级和数量（#id>class，html#id>#id），同级级别和数量一致，再看下一级属性。当然了几个基本原则还是列一下：[label]#ID > #ID > [label].class > .class > [label]。但是这是有前提的，我们要优先考虑的是样式是否具体描述到该元素。

来看几个例子:

<style type=”text/css”>
<!–

#id3{font-size:18px}
.class1 .class2 .class3{font-size:12px} / 描述的再具体也不起作用 — 原则二 /
.class3{font-size:18px}
div p span{font-size:12px}
–>
</style>
&lt;div class="class1"&gt;&lt;p class="class2"&gt; &lt;span class="class3" id="id3"&gt;我是多大字号？&lt;/span&gt; &lt;/p&gt;&lt;/div&gt;
运行结果：

#id3{font-size:18px}

如果删掉前两行的话，那么.class3 就会起作用。

其实说到CSS优先级就应该讨论到超链接访问过后hover样式就不出现的问题。这就是牵扯到CSS的覆盖问题了，后面定义的属性会覆盖前面的，所以我们在定义hover样式的时候一定要按照下面的顺序：L-V-H-A

<style type=”text/css”>
<!–
a:link {}
a:visited {}
a:hover {}
a:active {}
–>
</style>

这么书写之后就解决了超链接访问过后hover样式就不出现的问题。
