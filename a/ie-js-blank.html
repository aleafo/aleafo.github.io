IE对js中空格的识别问题

  2011-07-22     Javascript     兼容性

IE对js中的空格识别有问题。下面的语句

1. if (document.getElementById(‘maohao’) == ‘’)
2. if (document.getElementById(‘maohao’) == ‘ ‘)

这两句的效果在IE和其他浏览器中的判断是不一样的。在IE中会认为这两句语句是等效的，而在其他浏览器中会认为第2句中引号内部是一个空格。

示例，下面的冒号闪动效果，在IE中是无效的，而在Chrome或者Firefox中是正常的。

<script type=”text/javascript”>
  window.setInterval(function(){
      if (document.getElementById(“maohao”).innerHTML == “ “){
      document.getElementById(“maohao”).innerHTML =  “:”;
      }else{
      document.getElementById(“maohao”).innerHTML =” “;
      }
      },1000);
</script>
<div id=”maohao”></div>

修改为下面的代码即可正常执行

<script type=”text/javascript”>
  window.setInterval(function(){
      if (document.getElementById(“maohao”).innerHTML == “&nbsp;”){
      document.getElementById(“maohao”).innerHTML =  “:”;
      }else{
      document.getElementById(“maohao”).innerHTML =”&nbsp;”;
      }
      },1000);
</script>
<div id=”maohao”>&nbsp;</div>

所以为了兼容IE浏览器需要在js的编写过程中尽量不要使用空格进行判断的形式，否则在IE中将无法区分和识别。
